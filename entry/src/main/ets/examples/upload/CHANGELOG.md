# 更新日志

## 2025-12-31

### 修复
- 修复了 `crypto.subtle.digest` 在某些浏览器中不可用的问题
- 添加了降级方案：当 Crypto API 不可用时，自动使用简单哈希算法
- 改进了错误处理，文件处理失败时会显示详细错误信息

### 改进
- 增强了浏览器兼容性
- 添加了 `webkitSubtle` 支持（Safari 旧版本）
- 优化了哈希计算逻辑，提供更好的用户体验

### 技术细节

#### 哈希计算策略
1. **优先使用 Crypto API**：如果浏览器支持 `crypto.subtle.digest`，使用 SHA-256 算法
2. **降级方案**：如果不支持，使用简单的字符串哈希算法（非加密级别）
3. **兼容性检查**：同时检查 `crypto.subtle` 和 `crypto.webkitSubtle`（Safari）

#### 简单哈希算法
- 使用位运算实现的快速哈希函数
- 基于文件名、大小和修改时间生成唯一标识
- 输出64个字符的十六进制字符串（模拟SHA-256长度）
- 仅用于文件标识，不用于安全目的

### 浏览器支持
- ✅ Chrome 60+（完整支持）
- ✅ Firefox 55+（完整支持）
- ✅ Safari 11+（完整支持）
- ✅ Edge 79+（完整支持）
- ⚠️ 旧版浏览器（降级支持）

### 注意事项
- 简单哈希算法不是加密级别的哈希，仅用于文件标识
- 在生产环境中，建议使用支持 Crypto API 的现代浏览器
- 如果需要更强的安全性，可以考虑使用第三方库（如 crypto-js）
