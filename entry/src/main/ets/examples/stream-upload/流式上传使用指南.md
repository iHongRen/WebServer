# 流式上传使用指南

## 概述

本示例展示了WebServer框架如何接收客户端使用`Transfer-Encoding: chunked`进行的流式上传。流式上传允许客户端在不知道完整数据大小的情况下，以分块的方式传输数据，特别适合大文件上传和实时数据传输场景。

## 什么是分块传输编码（Chunked Transfer Encoding）

分块传输编码是HTTP/1.1协议中的一种数据传输机制，允许服务器或客户端在不知道内容总长度的情况下开始传输数据。

### 工作原理

1. **请求头设置**：客户端在HTTP请求头中设置`Transfer-Encoding: chunked`
2. **数据分块**：数据被分成多个块（chunk）进行传输
3. **块格式**：每个块包含：
   - 块大小（十六进制）+ `\r\n`
   - 块数据 + `\r\n`
4. **结束标记**：最后发送大小为0的块表示传输结束：`0\r\n\r\n`

### 示例格式

```
POST /api/upload HTTP/1.1
Host: example.com
Transfer-Encoding: chunked

5\r\n
Hello\r\n
6\r\n
 World\r\n
0\r\n
\r\n
```

## 框架支持

### 服务器端自动处理

WebServer框架已经内置了对分块传输编码的支持：

1. **HttpServer.isRequestComplete()**
   - 自动检测`Transfer-Encoding: chunked`头
   - 等待接收完整的分块数据（直到收到`0\r\n\r\n`结束标记）

2. **HttpRequest.parseBody()**
   - 自动解码分块传输的数据
   - 将分块数据合并为完整的请求体
   - 支持各种Content-Type的自动解析

### 使用方法

在路由处理函数中，您可以像处理普通请求一样处理流式上传：

```typescript
server.post('/api/upload', (req: HttpRequest, res: HttpResponse) => {
  // 检查是否使用了分块传输
  const isChunked = req.get('transfer-encoding')?.includes('chunked');
  
  // 获取解码后的完整数据
  const data = req.body;
  
  // 或获取原始ArrayBuffer
  const rawData = req.getRawBody();
  
  res.json({ success: true, isChunked: isChunked });
});
```

## 支持的上传类型

### 1. 流式文本上传

接收分块传输的文本数据：

```bash
curl -X POST http://IP:8087/api/upload/stream-text \
  -H "Transfer-Encoding: chunked" \
  -d "Hello, Streaming Upload!"
```

### 2. 流式JSON上传

接收分块传输的JSON数据：

```bash
curl -X POST http://IP:8087/api/upload/stream-json \
  -H "Transfer-Encoding: chunked" \
  -H "Content-Type: application/json" \
  -d '{"name":"test","value":123}'
```

### 3. 流式文件上传

接收并保存流式上传的文件：

```bash
curl -X POST http://IP:8087/api/upload/stream-file?filename=test.txt \
  -H "Transfer-Encoding: chunked" \
  --data-binary @file.txt
```

### 4. 大文件流式上传

支持大文件的流式上传：

```bash
curl -X POST http://IP:8087/api/upload/stream-large \
  -H "Transfer-Encoding: chunked" \
  --data-binary @large-file.bin
```

## API端点

**响应：**

**请求头：**

**响应：**

```json
  "isChunked": true,
```


## 使用场景

### 1. 大文件上传

当上传大文件时，使用分块传输可以：
- 避免一次性加载整个文件到内存
- 支持断点续传（配合Range头）
- 提供更好的用户体验

### 2. 实时数据流

适用于实时数据传输场景：
- 日志流上传
- 传感器数据上传
- 实时监控数据

### 3. 动态生成的内容

当内容是动态生成且大小未知时：
- 数据库查询结果流式上传
- 压缩数据流
- 加密数据流

## 客户端实现示例

### 使用curl

```bash
# 基本用法
curl -X POST http://IP:8087/api/upload/stream-text \
  -H "Transfer-Encoding: chunked" \
  -d "data"

# 上传文件
curl -X POST http://IP:8087/api/upload/stream-file \
  -H "Transfer-Encoding: chunked" \
  --data-binary @file.txt

# 管道输入
echo "data" | curl -X POST http://IP:8087/api/upload/stream-text \
  -H "Transfer-Encoding: chunked" \
  --data-binary @-
```

### 使用HarmonyOS HTTP客户端

```typescript
import { http } from '@kit.NetworkKit';

// 创建HTTP请求
let httpRequest = http.createHttp();

// 准备数据
let data = "Hello, Streaming Upload!";

// 发送请求
httpRequest.request(
  "http://192.168.2.38:8087/api/upload/stream-text",
  {

}).catch((error) => {
  console.error('Error:', error);
});
```

## 性能优化建议

### 1. 块大小选择

- 小块（4KB-8KB）：适合实时性要求高的场景
- 中块（64KB-256KB）：平衡性能和实时性
- 大块（1MB+）：适合大文件传输

### 2. 缓冲区管理

框架自动管理缓冲区，但您可以：
- 及时处理接收到的数据
- 避免在内存中保存过大的数据
- 使用流式写入文件

### 3. 错误处理

```typescript
server.post('/api/upload', async (req: HttpRequest, res: HttpResponse) => {
  try {
    const rawBody = req.getRawBody();
    
    // 检查数据大小
    if (rawBody.byteLength > MAX_SIZE) {
      return res.status(413).json({ error: '数据过大' });
    }
    
    // 处理数据
    await processData(rawBody);
    
    res.json({ success: true });
  } catch (error) {
    res.status(500).json({ error: error.message });
  }
});
```

## 与普通上传的对比

| 特性 | 普通上传 (Content-Length) | 流式上传 (Chunked) |
|------|--------------------------|-------------------|
| 数据大小 | 必须预先知道 | 可以未知 |
| 内存占用 | 需要完整缓冲 | 可以边接收边处理 |
| 实时性 | 等待完整数据 | 可以实时处理 |
| 断点续传 | 需要额外实现 | 天然支持 |
| 适用场景 | 小文件、已知大小 | 大文件、流式数据 |

## 注意事项

1. **HTTP版本**：分块传输编码需要HTTP/1.1或更高版本
2. **代理兼容性**：某些代理服务器可能不支持分块传输
3. **超时设置**：长时间的流式上传需要适当的超时配置
4. **错误恢复**：实现适当的错误处理和重试机制
