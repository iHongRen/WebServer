# 静态文件服务中间件说明

## 概述

静态文件服务中间件用于提供HTML、CSS、JavaScript、图片等静态资源。现在支持两种模式：
- **单站点模式**：传统的单一静态文件目录服务
- **多站点模式**：支持多个独立的静态站点，每个站点有独立配置

## 单站点模式（传统方式）

### 基本使用

假设你的静态文件存放在 `www` 目录下：

```
/
├── www/
│   ├── index.html
│   ├── styles.css
│   └── images/
│       └── logo.png
└── ...
```

你可以这样配置静态文件服务：

```typescript
import { HttpServer } from '@cxy/webserver';

const server = new HttpServer();

// 所有对根URL的请求都会尝试在 'www' 目录中查找文件
const root = getContext().filesDir + '/www';
server.serveStatic(root, {
  maxAge: 3600  // 缓存1小时
});
```

现在，你可以通过以下URL访问静态文件：
- `http://your-ip:8080/index.html`
- `http://your-ip:8080/styles.css`
- `http://your-ip:8080/images/logo.png`

## 多站点模式（新功能）

### 基本概念

多站点模式允许在同一个服务器上托管多个独立的静态网站，每个站点有：
- 独立的URL路径前缀（如 `/docs`, `/blog`, `/api`）
- 独立的根目录
- 独立的缓存策略
- 独立的索引文件配置
- 独立的目录浏览设置

### 使用方法

```typescript
import { HttpServer } from '@cxy/webserver';

const server = new HttpServer();

// 注册站点1：文档中心
server.registerStaticSite('/docs', '/path/to/docs', {
  maxAge: 3600,              // 1小时缓存
  index: ['index.html'],     // 索引文件
  directoryListing: false    // 禁用目录浏览
});

// 注册站点2：图片画廊
server.registerStaticSite('/gallery', '/path/to/gallery', {
  maxAge: 7200,              // 2小时缓存
  index: ['index.html'],
  directoryListing: true     // 启用目录浏览
});

// 注册站点3：API文档
server.registerStaticSite('/api-docs', '/path/to/api-docs', {
  maxAge: 1800,              // 30分钟缓存
  index: ['index.html', 'index.htm'],
  directoryListing: true
});

// 启用多站点服务（必须调用）
server.serveMultiSite();

await server.startServer(8080);
```

### 访问站点

```
http://your-ip:8080/docs/           -> 文档中心首页
http://your-ip:8080/docs/guide.html -> 文档中心的guide.html
http://your-ip:8080/gallery/        -> 图片画廊首页
http://your-ip:8080/gallery/photos/ -> 图片目录（支持浏览）
http://your-ip:8080/api-docs/       -> API文档首页
```

### 配置选项

| 选项 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `maxAge` | number | 3600 | 缓存时间（秒） |
| `index` | string[] | `['index.html', 'index.htm']` | 索引文件列表 |
| `directoryListing` | boolean | false | 是否启用目录浏览 |

### 目录浏览功能

当启用 `directoryListing: true` 时，访问目录会显示美观的文件列表：

```html
Index of /photos
┌─────────────────────────────────────┐
│ Name          Size      Modified    │
├─────────────────────────────────────┤
│ 📁 ..                               │
│ 📁 2024/      -         2024-01-15  │
│ 📄 photo1.jpg 2.5 MB    2024-01-10  │
│ 📄 photo2.jpg 3.1 MB    2024-01-12  │
└─────────────────────────────────────┘
```

### 站点管理API

```typescript
// 获取所有已注册的站点
const sites = server.getStaticSites();

// 注销站点
server.unregisterStaticSite('/docs');
```

### 路由匹配规则

多站点使用**最长前缀匹配**算法：

```
请求: /docs/guide.html
匹配: /docs 站点

请求: /docs/api/reference.html
匹配: /docs 站点（处理 /api/reference.html）

请求: /gallery/
匹配: /gallery 站点（查找索引文件）

请求: /other.html
不匹配任何站点，传递给下一个中间件
```

## 缓存策略

### 服务器端缓存
- 基于文件路径和ETag的缓存
- 自动缓存失效（文件修改时）
- 所有站点共享缓存池

### 客户端缓存
- 通过 `Cache-Control` 头控制
- 支持 `ETag` 验证
- 304 Not Modified 响应

### 缓存配置建议

```typescript
// 静态资源（长期缓存）
server.registerStaticSite('/assets', '/var/www/assets', {
  maxAge: 31536000  // 1年
});

// 动态内容（短期缓存）
server.registerStaticSite('/news', '/var/www/news', {
  maxAge: 300  // 5分钟
});

// 开发环境（不缓存）
server.registerStaticSite('/dev', '/var/www/dev', {
  maxAge: 0
});
```

## 使用场景

### 场景1：企业网站 + 文档中心

```typescript
server.registerStaticSite('/', '/var/www/main', {
  maxAge: 3600
});

server.registerStaticSite('/docs', '/var/www/docs', {
  maxAge: 7200,
  directoryListing: true
});

server.serveMultiSite();
```

### 场景2：多版本API文档

```typescript
server.registerStaticSite('/api/v1', '/docs/api/v1', {
  maxAge: 86400
});

server.registerStaticSite('/api/v2', '/docs/api/v2', {
  maxAge: 3600
});

server.serveMultiSite();
```

### 场景3：多语言网站

```typescript
server.registerStaticSite('/zh', '/var/www/zh', {
  maxAge: 3600
});

server.registerStaticSite('/en', '/var/www/en', {
  maxAge: 3600
});

server.serveMultiSite();
```
