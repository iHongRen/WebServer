# 多站点静态文件服务使用指南

## 概述

WebServer现在支持多静态子站点功能，允许在同一个服务器上托管多个独立的静态网站，每个站点有独立的：
- URL路径前缀
- 根目录
- 缓存策略
- 索引文件配置
- 目录浏览设置

## 核心特性

### 1. 多站点管理
- 支持注册多个静态站点
- 每个站点有独立的配置
- 自动按最长前缀匹配路由

### 2. 站点级配置
- **maxAge**: 缓存时间（秒）
- **index**: 索引文件列表（如 `['index.html', 'index.htm']`）
- **directoryListing**: 是否启用目录浏览

### 3. 性能优化
- 统一的缓存管理
- ETag支持
- 客户端缓存验证
- 最长前缀匹配算法

### 4. 安全性
- 路径遍历攻击防护
- 独立的站点隔离
- 可配置的目录浏览权限

## 使用方法

### 基本用法

```typescript
import { HttpServer } from '@cxy/webserver';

const server = new HttpServer();

// 注册站点1：文档中心
server.registerStaticSite('/docs', '/path/to/docs', {
  maxAge: 3600,              // 1小时缓存
  index: ['index.html'],     // 索引文件
  directoryListing: false    // 禁用目录浏览
});

// 注册站点2：图片画廊
server.registerStaticSite('/gallery', '/path/to/gallery', {
  maxAge: 7200,              // 2小时缓存
  index: ['index.html'],
  directoryListing: true     // 启用目录浏览
});

// 注册站点3：API文档
server.registerStaticSite('/api-docs', '/path/to/api-docs', {
  maxAge: 1800,              // 30分钟缓存
  index: ['index.html', 'index.htm'],
  directoryListing: true
});

// 启用多站点服务（必须调用）
server.serveMultiSite();

// 可选：添加主站点作为后备
server.serveStatic('/path/to/main-site', { maxAge: 0 });

await server.startServer(8080);
```

### 站点管理

```typescript
// 获取所有已注册的站点
const sites = server.getStaticSites();
console.log(sites);

// 注销站点
server.unregisterStaticSite('/docs');
```

### 路由匹配规则

多站点使用**最长前缀匹配**算法：

```
请求: /site1/page.html
匹配: /site1 站点

请求: /site1/sub/page.html
匹配: /site1 站点（处理 /sub/page.html）

请求: /site2/
匹配: /site2 站点（查找索引文件）

请求: /other.html
不匹配任何站点，传递给下一个中间件
```

## 缓存策略

### 服务器端缓存
- 所有站点共享统一的缓存池
- 基于文件路径和ETag的缓存键
- 自动缓存失效（文件修改时）

### 客户端缓存
- 通过 `Cache-Control` 头控制
- 支持 `ETag` 验证
- 304 Not Modified 响应

### 缓存配置建议

```typescript
// 静态资源（长期缓存）
server.registerStaticSite('/assets', '/var/www/assets', {
  maxAge: 31536000  // 1年
});

// 动态内容（短期缓存）
server.registerStaticSite('/news', '/var/www/news', {
  maxAge: 300  // 5分钟
});

// 开发环境（不缓存）
server.registerStaticSite('/dev', '/var/www/dev', {
  maxAge: 0
});
```

## 性能考虑

### 优化建议

1. **合理设置缓存时间**
   - 静态资源：长期缓存（1天-1年）
   - 动态内容：短期缓存（5分钟-1小时）
   - 开发环境：禁用缓存

2. **站点数量**
   - 建议不超过20个站点
   - 过多站点会影响路由匹配性能

3. **目录浏览**
   - 仅在必要时启用
   - 大目录会影响响应时间

4. **文件大小**
   - 大文件会占用缓存空间
   - 考虑使用CDN处理大文件

## 安全注意事项

1. **路径遍历防护**
   - 自动检测 `..` 路径
   - 返回400错误

2. **目录浏览权限**
   - 默认禁用目录浏览
   - 仅在安全的目录启用

3. **文件访问控制**
   - 确保站点根目录权限正确
   - 避免暴露敏感文件

## 故障排查

### 站点无法访问

1. 检查站点是否已注册
```typescript
console.log(server.getStaticSites());
```

2. 确认已调用 `serveMultiSite()`
```typescript
server.serveMultiSite();
```

3. 检查路径前缀是否正确
```typescript
// 正确
server.registerStaticSite('/site1', '/path/to/site1');

// 错误（缺少前导斜杠）
server.registerStaticSite('site1', '/path/to/site1');
```

### 缓存问题

清空缓存：
```typescript
StaticFiles.clearCache();
```

查看缓存统计：
```typescript
const stats = StaticFiles.getCacheStats();
console.log(stats);
```

### 目录浏览不工作

1. 确认已启用 `directoryListing: true`
2. 检查目录是否存在
3. 确认目录有读取权限
