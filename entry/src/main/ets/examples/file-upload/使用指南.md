# 文件上传使用指南

本示例覆盖三种上传方式，适用于不同场景：

| 方式 | 接口前缀 | 适用场景 |
|------|----------|----------|
| FileUpload 中间件 | `/api/upload/basic` 等 | 中小文件（< 100 MB），表单、多文件 |
| 流式上传 | `/api/upload/stream` | GB 级大文件，内存占用固定 |
| 分片上传 | `/api/upload/chunk/` | 超大文件、弱网断点续传 |

---

## 一、FileUpload 中间件

基于 `express-fileupload` 设计，整个请求体缓冲进内存后再处理，适合中小文件。

### 快速开始

```typescript
import { HttpServer, FileUpload } from '@cxy/webserver';
import type { UploadedFile } from '@cxy/webserver';

const server = new HttpServer();

server.use(FileUpload.create({
  tempFileDir: context.filesDir + '/temp',
  limits: { fileSize: 50 * 1024 * 1024 } // 50MB
}));

server.post('/upload', async (req, res) => {
  const file = req.files['file'] as UploadedFile;
  await file.mv(context.filesDir + '/uploads/' + file.name);
  res.json({ success: true, name: file.name, size: file.size });
});
```

### 配置选项

```typescript
FileUpload.create({
  createParentPath: true,       // 自动创建父目录
  uriDecodeFileNames: true,     // URI 解码文件名
  safeFileNames: true,          // 移除特殊字符
  abortOnLimit: true,           // 超限时抛出错误
  tempFileDir: '/tmp/uploads',  // 临时文件目录
  debug: false,
  limits: {
    fileSize: 100 * 1024 * 1024, // 单文件最大 100MB
    files: 10,                    // 最多 10 个文件
    fields: 100,                  // 最多 100 个表单字段
    fieldSize: 1024 * 1024        // 字段最大 1MB
  }
})
```

### UploadedFile 接口

```typescript
interface UploadedFile {
  name: string;          // 原始文件名
  data: ArrayBuffer;     // 文件数据
  size: number;          // 文件大小（字节）
  mimetype: string;      // MIME 类型
  tempFilePath: string;  // 临时文件路径
  truncated: boolean;    // 是否被截断
  mv: (path: string) => Promise<void>; // 移动到目标路径并删除临时文件
}
```

### API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/upload/basic`    | 基本上传（multipart/form-data） |
| POST | `/api/upload/multiple` | 多文件上传 |

## 二、流式上传

使用 `server.stream()` 注册路由，HTTP header 到达后立即触发路由处理，body 数据以 TCP 包为单位逐块推送给 `req.uploadStream`，直接写入磁盘。内存占用固定在单个 TCP 包大小（~4~64 KB），适合 GB 级大文件。

### 与 FileUpload 中间件对比

| | FileUpload 中间件 | 流式上传 |
|---|---|---|
| 触发时机 | 整个 body 进内存后 | header 到达后立即 |
| 内存占用 | 整个文件大小 | 单个 TCP 包（~64 KB） |
| 适合大小 | < 100 MB | 无限制，GB 级均可 |
| 进度感知 | 不支持 | `stream.progress` 实时进度 |

### 服务端代码

```typescript
// 注册流式路由（默认 POST，第三个参数可传 'PUT'）
server.stream('/api/upload/stream', async (req, res) => {
  const stream = req.uploadStream!;
  const fileName = req.get('x-file-name') ?? `upload_${Date.now()}.bin`;
  const destPath = `${uploadDir}/${fileName}`;

  const file = await fileIo.open(destPath,
    fileIo.OpenMode.CREATE | fileIo.OpenMode.WRITE_ONLY | fileIo.OpenMode.TRUNC);
  let fd = file.fd;
  let bytesWritten = 0;

  stream.on('data', async (chunk: ArrayBuffer) => {
    // 每个 TCP 包到达时触发，必须传 offset 保证顺序写入
    await fileIo.write(fd, chunk, { offset: bytesWritten, length: chunk.byteLength });
    bytesWritten += chunk.byteLength;
    console.log(`进度: ${(stream.progress * 100).toFixed(1)}%`);
  });

  stream.on('end', async () => {
    await fileIo.close(fd);
    res.json({ success: true, size: stream.bytesReceived });
  });

  stream.on('error', async (err: Error) => {
    await fileIo.close(fd).catch(() => {});
    await fileIo.unlink(destPath).catch(() => {});
    res.status(500).json({ error: err.message });
  });
});
```

### curl 示例

```bash
# 使用 -T 流式发送（不会把文件读入内存，支持 GB 级）
curl -X POST http://IP:8080/api/upload/stream \
     -H "Content-Type: application/octet-stream" \
     -H "X-File-Name: bigfile.bin" \
     -H "Expect:" \
     -T /path/to/large/file.bin
```

> **注意**：必须用 `-T`，不要用 `--data-binary @file`。  
> `--data-binary` 会把整个文件读入 curl 的内存，1 GB 文件会直接 OOM。  
> `-H "Expect:"` 清除 curl 默认的 `Expect: 100-continue`，避免上传前额外等待。

## 三、分片上传（断点续传）

客户端将大文件分割成小分片逐个上传，服务端保存分片，全部上传完成后合并。支持中断后根据已上传分片继续，不必重传。

### 流程

```
1. 客户端计算文件哈希
2. POST /api/upload/chunk/check  → 查询已上传分片（断点续传判断）
3. POST /api/upload/chunk/upload → 上传每个分片（form-data: chunk, chunkIndex, totalChunks, fileName, fileHash）
4. POST /api/upload/chunk/merge  → 所有分片上传完毕后合并
```

### API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST   | `/api/upload/chunk/check`          | 查询已上传分片 |
| POST   | `/api/upload/chunk/upload`         | 上传单个分片 |
| POST   | `/api/upload/chunk/merge`          | 合并所有分片 |
| DELETE | `/api/upload/chunk/cancel/:hash`   | 取消上传并清理分片 |
| GET    | `/api/upload/chunk/tasks`          | 查看进行中的任务 |

### curl 示例

```bash
# 1. 查询断点续传状态
curl -X POST http://IP:8080/api/upload/chunk/check \
  -H "Content-Type: application/json" \
  -d '{"fileHash": "abc123", "fileName": "test.zip"}'

# 2. 上传分片（chunkIndex 从 0 开始）
curl -X POST http://IP:8080/api/upload/chunk/upload \
  -F "chunk=@chunk_0.bin" \
  -F "chunkIndex=0" \
  -F "totalChunks=10" \
  -F "fileName=test.zip" \
  -F "fileHash=abc123"

# 3. 合并分片
curl -X POST http://IP:8080/api/upload/chunk/merge \
  -H "Content-Type: application/json" \
  -d '{"fileHash": "abc123", "fileName": "test.zip"}'
```

## 注意事项

1. **流式上传必须传 `offset`**：`fileIo.write(fd, chunk)` 默认从文件头写，多块时会互相覆盖，必须传 `{ offset: bytesWritten }`。
2. **分片上传的哈希**：用文件内容哈希（MD5/SHA）作为 `fileHash`，同一文件不同名也能复用已上传分片。
3. **临时文件清理**：调用 `file.mv()` 后临时文件自动删除；分片文件在合并后自动清理；停止服务器时会清理所有临时文件。
4. **大文件流式上传限制**：框架底层使用 HarmonyOS TCPSocket，单次 `message` 事件接收的数据量取决于系统 TCP 缓冲区，通常每次推送 16 KB ~ 64 KB。
