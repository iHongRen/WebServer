# 文件上传中间件使用指南

## 概述

文件上传中间件基于 express-fileupload 设计，使用鸿蒙 `fs.createWriteStream` API 实现流式文件上传处理。

## 核心特性

### 1. 流式处理
- 使用 `fs.createWriteStream` 进行流式写入
- 边接收边写入磁盘，内存占用低
- 适合处理大文件上传

### 2. 多格式支持
- `multipart/form-data`: 标准表单文件上传
- `application/octet-stream`: 二进制流上传

### 3. 临时文件管理
- 自动创建临时文件
- 提供 `mv()` 方法移动文件
- 移动后自动清理临时文件

### 4. 灵活配置
- 文件大小限制
- 文件数量限制
- 字段数量和大小限制
- 安全文件名处理
- 自动创建目录

## 快速开始

### 基本使用

```typescript
import { HttpServer, FileUpload } from '@cxy/webserver';
import type { UploadedFile } from '@cxy/webserver';

const server = new HttpServer();

server.post('/upload',
  FileUpload.create({
    tempFileDir: context.filesDir + '/temp',
    limits: {
      fileSize: 10 * 1024 * 1024 // 10MB
    }
  }),
  async (req, res) => {
    const file = req.files['file'] as UploadedFile;
    
    // 移动文件到目标位置
    await file.mv(context.filesDir + '/uploads/' + file.name);
    
    res.json({
      success: true,
      file: {
        name: file.name,
        size: file.size,
        mimetype: file.mimetype
      }
    });
  }
);
```

## 配置选项详解

### FileUploadOptions

```typescript
interface FileUploadOptions {
  createParentPath?: boolean;      // 自动创建父目录
  uriDecodeFileNames?: boolean;    // URI 解码文件名
  safeFileNames?: boolean;         // 安全文件名处理
  preserveExtension?: boolean;     // 保留文件扩展名
  abortOnLimit?: boolean;          // 超限时中止
  useTempFiles?: boolean;          // 使用临时文件
  tempFileDir?: string;            // 临时文件目录
  debug?: boolean;                 // 调试模式
  limits?: FileLimits;             // 文件限制
}

interface FileLimits {
  fileSize?: number;    // 单文件最大大小（字节）
  files?: number;       // 最大文件数量
  fields?: number;      // 最大字段数量
  fieldSize?: number;   // 字段最大大小
}
```

#### 1. 基本配置
```typescript
FileUpload.create({
```

#### 2. 安全配置

```typescript
})
```

## UploadedFile 接口

### 属性说明

```typescript
interface UploadedFile {
  name: string;              // 原始文件名
  data: ArrayBuffer;         // 文件数据
  size: number;              // 文件大小（字节）
  encoding: string;          // 编码方式
  tempFilePath: string;      // 临时文件路径
  truncated: boolean;        // 是否被截断
  mimetype: string;          // MIME类型
  md5?: string;              // MD5哈希（可选）
  mv: (path: string) => Promise<void>; // 移动文件方法
}
```

### 使用示例

```typescript
const file = req.files['avatar'] as UploadedFile;

// 获取文件信息
console.log('文件名:', file.name);
console.log('大小:', file.size, 'bytes');
console.log('类型:', file.mimetype);
console.log('临时路径:', file.tempFilePath);
console.log('是否截断:', file.truncated);

// 移动文件
const destPath = `/uploads/${file.name}`;
await file.mv(destPath);
```

## 使用场景

### 场景1: 单文件上传

```typescript
server.post('/upload/single',
  FileUpload.create({
    tempFileDir: tempDir,
    limits: { fileSize: 10 * 1024 * 1024 }
  }),
  async (req, res) => {
    const file = req.files['file'] as UploadedFile;
    
    if (!file) {
      res.status(400).json({ error: '没有上传文件' });
      return;
    }
    
    await file.mv(`/uploads/${file.name}`);
    res.json({ success: true, file: file.name });
  }
);
```

### 场景2: 多文件上传

```typescript
server.post('/upload/multiple',
  FileUpload.create({
    tempFileDir: tempDir,
    limits: { 
      fileSize: 5 * 1024 * 1024,
      files: 10
    }
  }),
  async (req, res) => {
    const files = req.files;
    const uploadedFiles: string[] = [];
    
    for (const key in files) {
      const file = files[key] as UploadedFile;
      await file.mv(`/uploads/${file.name}`);
      uploadedFiles.push(file.name);
    }
    
    res.json({ 
      success: true, 
      count: uploadedFiles.length,
      files: uploadedFiles 
    });
  }
);
```

### 场景3: 图片上传（带验证）

```typescript
server.post('/upload/image',
  FileUpload.create({
    tempFileDir: tempDir,
    limits: { fileSize: 5 * 1024 * 1024 }
  }),
  async (req, res) => {
    const file = req.files['image'] as UploadedFile;
    
    // 验证文件类型
    const allowedTypes = ['image/jpeg', 'image/png', 'image/gif'];
    if (!allowedTypes.includes(file.mimetype)) {
      res.status(400).json({ error: '只允许上传图片文件' });
      return;
    }
    
    // 生成新文件名
    const ext = file.name.substring(file.name.lastIndexOf('.'));
    const newName = `${Date.now()}${ext}`;
    
    await file.mv(`/uploads/images/${newName}`);
    res.json({ 
      success: true, 
      url: `/images/${newName}` 
    });
  }
);
```

### 场景4: 带表单字段的上传

```typescript
server.post('/upload/with-fields',
  FileUpload.create({
    tempFileDir: tempDir
  }),
  async (req, res) => {
    const file = req.files['file'] as UploadedFile;
    const fields = req.body as Record<string, string>;
    
    // 获取表单字段
    const title = fields.title;
    const description = fields.description;
    
    // 保存文件
    await file.mv(`/uploads/${file.name}`);
    
    // 保存元数据到数据库
    // await saveFileMetadata({ title, description, filename: file.name });
    
    res.json({ 
      success: true,
      file: file.name,
      metadata: { title, description }
    });
  }
);
```

### 场景5: 流式上传（application/octet-stream）

```typescript
server.post('/upload/stream',
  FileUpload.create({
    tempFileDir: tempDir,
    limits: { fileSize: 50 * 1024 * 1024 }
  }),

      success: true,

  }
);

//   -H "Content-Type: application/octet-stream" \
```

## 错误处理

### 基本错误处理

```typescript
server.post('/upload',
  FileUpload.create({ /* options */ }),
  async (req, res) => {
    try {
      const file = req.files['file'] as UploadedFile;
      await file.mv('/uploads/' + file.name);
      res.json({ success: true });
    } catch (error: Error) {
      console.error('上传失败:', error);
      res.status(500).json({
        success: false,
        error: error.message
      });
    }
  }
);
```

#### 1. 文件大小超限

```typescript
// 在服务器关闭时清理临时文件
async function cleanupTempFiles(tempDir: string) {
  try {
    const files = await fs.listFile(tempDir);
    for (const file of files) {
      await fs.unlink(`${tempDir}/${file}`);
    }
  } catch (error: Error) {
    console.error('清理失败:', error);
  }
}

// 在 stop() 方法中调用
await cleanupTempFiles(this.tempDir);
```

## 常见问题

### Q1: 如何获取上传进度？

A: 当前实现不支持实时进度，因为数据在到达服务器时已经完整接收。如需进度显示，建议：
- 客户端使用分片上传
- 使用 WebSocket 实时通知
- 参考分片上传示例

### Q2: 如何处理大文件上传？

A: 
1. 增加文件大小限制
2. 使用流式上传（application/octet-stream）
3. 考虑使用分片上传
4. 设置合理的超时时间

### Q3: 临时文件何时被删除？

A: 调用 `mv()` 方法后，临时文件会自动删除。如果没有调用 `mv()`，需要手动清理。

### Q4: 如何自定义文件名？

A:
```typescript
const file = req.files['file'] as UploadedFile;
const newName = `${Date.now()}_${file.name}`;
await file.mv(`/uploads/${newName}`);
```

### Q5: 如何限制特定文件类型？

A:
```typescript
const allowedTypes = ['image/jpeg', 'image/png'];
if (!allowedTypes.includes(file.mimetype)) {
  res.status(400).json({ error: '不支持的文件类型' });
  return;
}
```
