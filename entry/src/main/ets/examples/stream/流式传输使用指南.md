# 流式传输使用指南

## 1. 概述

流式传输（Streaming）是一种在数据生成时就开始发送数据的技术，而不是等待所有数据准备好后再一次性发送。这种方式特别适合处理大文件、实时数据、长时间运行的任务等场景。

WebServer 框架现在支持完整的 HTTP 流式传输功能，包括：
- 分块传输编码（Transfer-Encoding: chunked）
- Server-Sent Events (SSE)
- 文件流式传输
- 实时数据推送

## 2. 核心API

### 2.1. write() - 写入数据块

```typescript
await res.write(chunk: string | ArrayBuffer): Promise<boolean>
```

写入一个数据块到响应流中。

**参数**:
- `chunk`: 要发送的数据（字符串或ArrayBuffer）

**返回值**:
- `Promise<boolean>`: 是否成功写入

**示例**:
```typescript
server.get('/stream', async (req, res) => {
  res.setHeader('Content-Type', 'text/plain');
  
  for (let i = 1; i <= 10; i++) {
    await res.write(`数据块 ${i}\n`);
    await sleep(1000); // 延迟1秒
  }
  
  await res.end();
});
```

### 2.2. end() - 结束响应

```typescript
await res.end(chunk?: string | ArrayBuffer): Promise<void>
```

结束响应流。可以选择性地发送最后一块数据。

**参数**:
- `chunk`: 可选的最后一块数据

**示例**:
```typescript
await res.write('Hello ');
await res.write('World');
await res.end('!\n');
```

### 2.3. 分块传输编码

当使用 `write()` 和 `end()` 时，如果没有设置 `Content-Length`，框架会自动使用分块传输编码（`Transfer-Encoding: chunked`）。

你也可以显式设置：
```typescript
res.setHeader('Transfer-Encoding', 'chunked');
```

## 3. 使用场景

### 3.1. 流式文本数据

适用于逐步生成的文本内容，如日志、报告等。

```typescript
server.get('/api/stream/text', async (req, res) => {
  res.setHeader('Content-Type', 'text/plain; charset=utf-8');
  
  for (let i = 1; i <= 10; i++) {
    await res.write(`数据块 ${i}\n`);
    await sleep(500);
  }
  
  await res.end('完成\n');
});
```

**测试**:
```bash
curl -N http://localhost:8080/api/stream/text
```

### 3.2. Server-Sent Events (SSE)

用于服务器向客户端推送实时事件。

```typescript
server.get('/api/stream/events', async (req, res) => {
  res.setHeader('Content-Type', 'text/event-stream');
  res.setHeader('Cache-Control', 'no-cache');
  res.setHeader('Connection', 'keep-alive');
  
  for (let i = 1; i <= 5; i++) {
    const event = `data: ${JSON.stringify({
      id: i,
      message: `事件 ${i}`,
      timestamp: Date.now()
    })}\n\n`;
    
    await res.write(event);
    await sleep(1000);
  }
  
  await res.end();
});
```

**客户端代码**:
```javascript
const eventSource = new EventSource('http://localhost:8080/api/stream/events');

eventSource.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log('收到事件:', data);
};
```

### 3.3. 大文件流式传输

避免一次性加载整个文件到内存。

```typescript
server.get('/api/stream/file', async (req, res, next) => {
  try {
    const filePath = '/path/to/large/file.txt';
    const stat = await fileIo.stat(filePath);
    
    res.setHeader('Content-Type', 'text/plain');
    res.setHeader('Content-Length', stat.size.toString());
    
    const file = await fileIo.open(filePath, fileIo.OpenMode.READ_ONLY);
    
    try {
      const chunkSize = 4096; // 4KB
      let offset = 0;
      
      while (offset < stat.size) {
        const readSize = Math.min(chunkSize, stat.size - offset);
        const buffer = new ArrayBuffer(readSize);
        
        await fileIo.read(file.fd, buffer, { offset });
        await res.write(buffer);
        
        offset += readSize;
      }
      
      await res.end();
    } finally {
      await fileIo.close(file.fd);
    }
  } catch (error) {
    next(error);
  }
});
```

### 3.4. 实时日志流

模拟实时日志输出。

```typescript
server.get('/api/stream/logs', async (req, res) => {
  res.setHeader('Content-Type', 'text/plain');
  
  const logs = [
    '[INFO] 服务器启动',
    '[INFO] 加载配置',
    '[INFO] 连接数据库',
    '[INFO] 服务器就绪'
  ];
  
  for (const log of logs) {
    const timestamp = new Date().toISOString();
    await res.write(`[${timestamp}] ${log}\n`);
    await sleep(800);
  }
  
  await res.end();
});
```

### 3.5. 进度报告

实时报告长时间运行任务的进度。

```typescript
server.get('/api/stream/progress', async (req, res) => {
  res.setHeader('Content-Type', 'application/json');
  
  const totalSteps = 10;
  
  for (let step = 1; step <= totalSteps; step++) {
    const progress = {
      step,
      total: totalSteps,
      percentage: (step / totalSteps * 100).toFixed(2),
      message: `处理步骤 ${step}/${totalSteps}`
    };
    
    await res.write(JSON.stringify(progress) + '\n');
    await sleep(1000);
  }
  
  await res.end();
});
```

## 4. 最佳实践

### 4.1. 设置正确的响应头

```typescript
// 文本流
res.setHeader('Content-Type', 'text/plain; charset=utf-8');

// JSON流
res.setHeader('Content-Type', 'application/json');

// SSE
res.setHeader('Content-Type', 'text/event-stream');
res.setHeader('Cache-Control', 'no-cache');
res.setHeader('Connection', 'keep-alive');

// 禁用缓存
res.setHeader('Cache-Control', 'no-cache');
res.setHeader('X-Accel-Buffering', 'no'); // Nginx
```

### 4.2. 错误处理

```typescript
server.get('/stream', async (req, res, next) => {
  try {
    res.setHeader('Content-Type', 'text/plain');
    
    for (let i = 0; i < 10; i++) {
      const success = await res.write(`数据 ${i}\n`);
      if (!success) {
        throw new Error('写入失败');
      }
    }
    
    await res.end();
  } catch (error) {
    console.error('流式传输错误:', error);
    // 如果响应头已发送，无法再发送错误响应
    if (!res.isHeadersSent()) {
      next(error);
    }
  }
});
```

### 4.3. 资源清理

```typescript
server.get('/stream/file', async (req, res, next) => {
  let file = null;
  
  try {
    file = await fileIo.open(filePath, fileIo.OpenMode.READ_ONLY);
    
    // 流式传输...
    
  } catch (error) {
    next(error);
  } finally {
    // 确保文件被关闭
    if (file) {
      await fileIo.close(file.fd);
    }
  }
});
```

### 4.4. 控制流速

```typescript
// 避免发送过快导致内存问题
const CHUNK_SIZE = 4096;
const DELAY_MS = 10;

for (let i = 0; i < totalChunks; i++) {
  await res.write(chunk);
  await sleep(DELAY_MS); // 控制发送速度
}
```

## 5. 与普通响应的区别

| 特性 | 普通响应 (send/json) | 流式响应 (write/end) |
|------|---------------------|---------------------|
| 数据发送 | 一次性发送 | 分块发送 |
| 内存占用 | 需要完整数据在内存 | 只需当前块在内存 |
| Content-Length | 必须设置 | 可选（使用chunked） |
| 适用场景 | 小数据、静态内容 | 大文件、实时数据 |
| 客户端接收 | 等待完整响应 | 逐步接收 |

## 6. 注意事项

1. **响应头只能发送一次**: 一旦调用 `write()` 或 `send()`，响应头就会被发送，之后无法修改。

2. **错误处理限制**: 如果响应头已发送，无法再发送错误状态码，只能关闭连接。

3. **连接保持**: 流式传输期间，连接会一直保持打开状态。

4. **客户端支持**: 确保客户端支持分块传输编码（现代浏览器和HTTP客户端都支持）。

5. **代理和缓存**: 某些代理服务器可能会缓冲流式响应，需要设置适当的响应头。

6. **超时设置**: 长时间的流式传输可能触发超时，需要适当配置。

## 8. 性能优化

### 8.1. 合理的块大小

```typescript
// 太小：频繁的系统调用，性能差
const CHUNK_SIZE = 100; // ❌

// 太大：内存占用高，响应延迟
const CHUNK_SIZE = 10 * 1024 * 1024; // ❌

// 推荐：4KB - 64KB
const CHUNK_SIZE = 4096; // ✅
```

### 8.2. 批量写入

```typescript
// 不好：每次写入很小的数据
for (let i = 0; i < 1000; i++) {
  await res.write(`${i}\n`); // ❌
}

// 好：批量累积后写入
let buffer = '';
for (let i = 0; i < 1000; i++) {
  buffer += `${i}\n`;
  if (buffer.length >= 4096) {
    await res.write(buffer);
    buffer = '';
  }
}
if (buffer) {
  await res.write(buffer);
}
```

## 9. 常见问题

### Q1: 为什么客户端没有立即收到数据？

A: 可能是代理服务器或客户端缓冲了数据。尝试：
- 设置 `Cache-Control: no-cache`
- 设置 `X-Accel-Buffering: no`（Nginx）
- 使用 `curl -N` 禁用缓冲

### Q2: 如何知道客户端断开连接？

A: 当 `write()` 返回 `false` 或抛出异常时，表示连接已断开。

### Q3: 流式传输会影响性能吗？

A: 对于大文件和实时数据，流式传输实际上能提高性能，因为：
- 降低内存占用
- 减少首字节时间（TTFB）
- 提供更好的用户体验

### Q4: 可以同时使用 send() 和 write() 吗？

A: 不可以。一旦使用了其中一个，就不能再使用另一个。

## 10. 总结

流式传输是处理大数据和实时数据的强大工具。WebServer 框架提供了简单易用的API，让你可以轻松实现各种流式传输场景。

关键要点：
- ✅ 使用 `write()` 和 `end()` 进行流式传输
- ✅ 自动支持分块传输编码
- ✅ 适合大文件、实时数据、SSE等场景
- ✅ 注意错误处理和资源清理
- ✅ 设置正确的响应头
